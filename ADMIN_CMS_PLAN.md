# GitHub 기반 정적 CMS 구축 계획 (ADMIN_CMS_PLAN.md)

이 문서는 외부 데이터베이스(Firebase 등) 없이 GitHub API를 사용하여 홈페이지 콘텐츠를 직접 관리하는 정적 CMS의 단계별 구현 계획을 설명합니다.

## 1단계: 기반 마련 및 데이터 읽기 (현재 완료)
- **데이터 구조 전환**: 모든 TypeScript 데이터 파일을 JSON 형식으로 전환하여 관리자 페이지에서 수정하기 용이한 구조로 변경했습니다.
- **GitHub 서비스(githubService.ts) 구축**: GitHub REST API를 호출하여 파일 내용을 읽어오고 토큰을 검증하는 기본 서비스를 구현했습니다.
- **보안 인증 구현**: GitHub Fine-grained Personal Access Token을 통해 관리자를 인증합니다. 토큰은 세션 메모리(`sessionStorage`)에만 보관하며 코드나 로컬 저장소에 남기지 않습니다.
- **읽기 전용 대시보드**: 저장소에서 실시간으로 JSON 데이터를 읽어와 요약 정보를 표시하는 관리자 화면을 구현했습니다.

## 2단계: 데이터 편집 및 저장 기능 (예정)
- **CRUD 인터페이스**: 매물, 후기, 오사카 정보의 등록, 수정, 삭제 폼을 다시 활성화합니다.
- **임시 상태 관리**: 수정 사항은 즉시 GitHub에 반영되지 않고 브라우저 내에서 임시로 보관됩니다.
- **통합 커밋 시스템**: 여러 파일의 수정사항을 모아 "사이트에 반영하기" 버튼을 클릭했을 때 GitHub의 Git Data API(Trees, Commits)를 사용하여 단 하나의 커밋으로 묶어 저장합니다. 이를 통해 Cloudflare Pages 배포를 1회로 최소화합니다.

## 3단계: 이미지 관리 및 최적화 (예정)
- **이미지 직접 업로드**: GitHub 저장소의 특정 폴더(`public/assets/properties/` 등)에 이미지를 직접 저장하는 기능을 추가합니다.
- **브라우저 측 압축 스택**: 업로드 전 브라우저에서 이미지 리사이즈 및 WebP/JPEG 압축을 수행하여 저장소 용량 낭비를 방지합니다.
- **파일명 규칙 자동화**: 매물 ID와 연동된 체계적인 파일명 규칙을 적용합니다.

## GitHub 토큰 권한 설정 가이드
관리자 페이지 로그인을 위해 생성할 GitHub PAT(Fine-grained) 권한은 다음과 같이 최소화하여 설정하는 것을 권장합니다.
1. **Repository selection**: 'Only select repositories' 선택 후 본 프로젝트 저장소 지정
2. **Permissions (Contents)**: `Access: Read and write` 설정
3. **만료일**: 보안을 위해 적절한 기간(예: 90일) 설정 권장
