
# 오사카J부동산 관리자 CMS - 이미지 업로드 가이드 (Stage 3)

본 문서는 Stage 3에서 도입된 이미지 업로드 및 관리 기능에 대한 가이드입니다.

## 1. 이미지 저장 위치
모든 이미지는 GitHub 저장소 내부의 `public/assets/uploads/` 폴더에 분류되어 저장됩니다.

- **매물 이미지**: `public/assets/uploads/properties/{propertyId}/`
- **후기 이미지**: `public/assets/uploads/reviews/`
- **오사카 정보**: `public/assets/uploads/osaka-info/`

## 2. 이미지 압축 및 리사이즈 규칙
웹사이트 성능을 최적화하기 위해 업로드 시 브라우저에서 자동으로 압축 및 리사이즈를 수행합니다.

- **포맷**: 모든 이미지는 `WebP` 형식으로 변환됩니다.
- **매물/정보글**: 긴 변 기준 최대 `1600px`, 품질 `0.8`.
- **후기**: 긴 변 기준 최대 `800px`, 품질 `0.8`.
- **용량 제한**: 원본 파일 기준 `10MB`를 초과하는 파일은 업로드가 제한됩니다.

## 3. 매물 이미지 관리
매물 편집 화면에서 매물 이미지를 자유롭게 관리할 수 있습니다.

- **다중 업로드**: 한 매물당 최대 **15장**까지 업로드 가능합니다.
- **순서 변경**: 목록의 핸들(드래그 아이콘)을 잡고 위아래로 끌어다 놓으면 데이터 반영 순서가 변경됩니다.
- **대표 이미지**: 가장 첫 번째(맨 위)에 위치한 이미지가 자동으로 대표 이미지(Cover)로 지정됩니다.
- **삭제**: 삭제 버튼을 누르면 목록에서 제거되며, "사이트에 반영하기" 시 실제 GitHub 파일 삭제도 함께 이루어집니다.

## 4. 후기 및 정보글 이미지
- 각 항목당 **1장**의 대표 이미지만 허용됩니다.
- 이미지 교체 또는 삭제가 가능합니다.

## 5. GitHub 단일 커밋 저장 방식
"사이트에 반영하기" 버튼을 클릭하면 아래 작업이 **단 1개의 GitHub 커밋**으로 처리됩니다.

1. 수정된 JSON 데이터 (매물, 후기, 정보 등) 반영
2. 새로 업로드 대기 중인 이미지 파일들을 GitHub에 추가
3. 삭제 대기 중인 이미지 파일들을 GitHub에서 물리적으로 제거

이 방식을 통해 Cloudflare Pages 배포는 단 한 번만 발생하며, 데이터와 이미지가 항상 동기화된 상태로 배포됩니다.

## 6. 주의 사항
- **용량 관리**: GitHub에 직접 이미지 파일을 저장하므로, 장기적으로 저장소 용량이 커질 수 있습니다. 가급적 불필요한 고화질 원본보다는 압축된 상태를 유지해 주세요.
- **삭제 정책**: CMS에서 이미지를 삭제하면 공개 사이트에서는 사라지지만, Git의 특성상 과거 커밋 히스토리에는 파일이 남아 있을 수 있습니다.
- **외부 URL**: 기존에 등록된 외부 사이트 이미지(HTTP 링크 등)는 삭제 시 JSON 경로만 제거되며, 외부 서버의 실제 파일이 삭제되지는 않습니다.
