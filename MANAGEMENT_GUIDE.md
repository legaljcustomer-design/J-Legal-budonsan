# 오사카J부동산 홈페이지 운영 및 관리 가이드

> **[중요 알림] 이 프로젝트는 외부 DB(Firebase 등)나 별도의 백엔드 서버 없이 100% 정적 사이트로만 운영됩니다. 향후 어떠한 경우에도 Firebase나 백엔드 아키텍처를 재도입하는 것을 엄격히 금지합니다.**

이 프로젝트는 현재 **Firebase(데이터베이스) 의존성을 완전히 제거**하고, 모든 데이터를 소스 코드 내의 정적 파일로 관리하는 **정적 홈페이지 구조**로 전환되었습니다. 이를 통해 데이터베이스 트래픽 초과(Quota Exceeded) 오류를 근본적으로 방지하고, 로딩 속도를 개선했습니다.

## 1. 주요 관리 방법 (데이터 수정)

이제 관리자 페이지 로그인 방식이 아닌, `src/data` 폴더 내의 파일을 수정하는 방식으로 홈페이지 내용을 관리합니다. AI Studio에서 다음과 같이 요청하시면 됩니다.

### A. 매물 추가 및 수정
- **파일 경로:** `src/data/properties.ts`
- **방법:** AI Studio에 "매물 하나 추가해줘"라고 말하고 사진, 가격, 위치 등의 정보를 전달하세요.
- **포인트:** 새로운 매물을 추가하면 자동으로 '매물 리스트' 상단에 노출됩니다.

### B. 고객 후기 관리
- **파일 경로:** `src/data/reviews.ts`
- **방법:** AI Studio에 "새로운 고객 후기 추가해줘"라고 요청하고 이미지 URL과 제목을 전달하세요.

### C. 오사카 정보 게시글 관리
- **파일 경로:** `src/data/osakaInfo.ts`
- **방법:** AI Studio에 "오사카 정보 게시글 내용 수정해줘"라고 요청하세요.

### D. 사이트 기본 정보 (연락처, 링크 등)
- **파일 경로:** `src/data/siteConfig.ts`
- **방법:** 인스타그램 링크, 카카오톡 ID, 히어로 섹션 문구 등을 이 파일에서 통합 관리합니다.

---

## 2. 현재 운영 방식의 특징

1. **Firebase 미사용:** 더 이상 Firestore나 Firebase Auth를 사용하지 않으므로, 월간 읽기 한도나 로그인 오류 걱정이 없습니다.
2. **관리자 페이지 비활성화:** 기존의 `/admin` 경로는 현재 메인 페이지로 자동 리다이렉트되도록 설정되어 일반 사용자에게 노출되지 않습니다.
3. **데이터 정적화:** 모든 매물과 설정은 파일로 저장되어 있으므로, 검색 엔진 최적화(SEO)에도 유리하며 서비스 안정성이 매우 높습니다.

---

## 3. 배포 흐름

1. **AI Studio 수정:** 채팅을 통해 매물 추가나 문구 수정을 요청합니다.
2. **GitHub 반영:** 수정된 코드가 GitHub 저장소에 push 됩니다.
3. **자동 배포:** 연결된 Cloudflare Pages에서 변경 사항을 감지하여 수 분 내에 실제 홈페이지에 반영합니다.

---

## 4. 제거된 구성 요소
- `firestore.rules`: 더 이상 사용되지 않음
- Firebase SDK 런타임 호출 (Firestore, Auth)
- 기존 브라우저 캐시 및 폴백 로직 (근본적인 데이터 구조 변경으로 불필요해짐)
