# 프로젝트 운영 및 기술 원칙 (PROJECT_RULES.md)

이 문서는 이 프로젝트의 영구적인 운영 원칙을 정의합니다. 앞으로의 모든 수정 및 기능 추가 작업은 반드시 이 원칙을 최우선으로 준수해야 합니다.

## 1. 운영 환경 및 도구
이 프로젝트는 오직 다음 3가지 요소만을 사용하여 운영합니다.
1. **Google AI Studio**: 코드 수정 및 데이터 관리 도구
2. **GitHub**: 소스 코드 저장소
3. **Cloudflare Pages**: 웹 호스팅 및 배포 플랫폼

## 2. 기술 아키텍처 (정적 사이트 우선)
이 프로젝트는 **100% 정적 프론트엔드 사이트(Static Site)** 구조를 유지합니다.

### [절대 금지 사항]
다음과 같은 외부 서버, 데이터베이스, 백엔드 서비스를 새로 도입하거나 복구하는 것을 엄격히 금지합니다.
- **Firebase 관련**: Firestore, Authentication, Realtime DB, Cloud Functions 등
- **기타 DB/백엔드**: Supabase, MongoDB, MySQL, PostgreSQL, 전용 API 서버 등
- **엣지 컴퓨팅/스토리지**: Cloudflare Workers, Pages Functions, D1, KV, R2 등
- **동적 기능**: 회원가입, 로그인, 관리자 인증 시스템, 실시간 데이터 조회 등

### [준수 사항]
- 모든 데이터는 프로젝트 내부의 정적 파일(`src/data/`)에서 관리합니다.
- 방문자가 페이지에 접속할 때 외부 서버나 DB를 호출하지 않아야 합니다.
- 모든 화면 구성 요소는 빌드 타임에 결정된 정적 에셋(HTML/CSS/JS)으로 완성되어야 합니다.

## 3. 데이터 및 콘텐츠 관리
- **매물/맨션**: `src/data/properties.ts`
- **고객 후기**: `src/data/reviews.ts`
- **지역 정보**: `src/data/osakaInfo.ts`
- **사이트 설정**: `src/data/siteConfig.ts`
- **이미지**: 프로젝트 내부 에셋 폴더에 저장하거나 고정된 정적 URL을 사용합니다. 이미지 추가 시 파일명을 정리하고 매물별로 폴더 구조를 정돈합니다.

## 4. 운영 방식
- 관리자 페이지(온라인 GUI)를 통한 데이터 수정은 하지 않습니다.
- 모든 업데이트(매물 추가, 문구 수정 등)는 Google AI Studio를 통해 해당 정적 데이터 파일을 직접 수정하거나 AI에게 수정을 요청하는 방식으로 진행합니다.
- 수정 사항은 GitHub 반영 후 Cloudflare Pages를 통해 자동 배포됩니다.
